KSP CAD/MDT — URUCHOMIENIE ONLINE NA RENDER
============================================

SYSTEM
- panel policyjny dla grupy RP na ER:LC,
- logowanie i weryfikacja roli przez Discord,
- kartoteki, mandaty, zatrzymania, raporty i zgłoszenia,
- aktywna służba z callsignem, radiowozem, partnerem i kanałem,
- centrum CAD z przydzielaniem patroli i zmianą statusów,
- opcjonalne powiadomienia Discord oraz rola „na służbie”.

DLACZEGO NIE GITHUB PAGES?
GitHub Pages wyświetla tylko statyczne HTML/CSS/JS. Ten projekt wymaga serwera Node.js,
więc musi działać na Renderze albo innym hostingu obsługującym Node.

CO WRZUCIĆ DO REPOZYTORIUM
- server.js
- package.json
- package-lock.json
- render.yaml
- .gitignore

WDROŻENIE
1. Wrzuć pliki do głównego katalogu repozytorium.
2. Na Render wybierz New -> Blueprint i połącz repozytorium.
3. Ustaw DISCORD_CLIENT_SECRET.
4. W Discord Developer Portal -> OAuth2 -> Redirects dodaj dokładnie:
   https://TWOJ-ADRES.onrender.com/auth/callback
5. Otwieraj stronę z adresu onrender.com.

WYMAGANE ZMIENNE
DISCORD_CLIENT_ID       ID aplikacji Discord.
DISCORD_CLIENT_SECRET   Sekret OAuth2. Nigdy nie umieszczaj go w publicznym repozytorium.
DISCORD_GUILD_ID        ID serwera Discord.
DISCORD_ROLE_ID         Rola dająca dostęp do systemu.
SESSION_SECRET          Długi losowy sekret sesji.

ZALECANA ZMIENNA ADMINISTRATORA
DISCORD_ADMIN_ROLE_ID   Rola pozwalająca usuwać rekordy.

Po zmianie bezpieczeństwa brak tej wartości NIE daje wszystkim administratora.
Osoby z rolą dostępową mogą korzystać z terminala, ale nie mogą trwale usuwać rekordów.

OPCJONALNE POŁĄCZENIE Z DISCORDEM
DISCORD_LOG_WEBHOOK_URL
Webhook kanału, na który system wysyła rozpoczęcia i zakończenia służby,
nowe zgłoszenia oraz przydziały patroli.

DISCORD_BOT_TOKEN
Token bota używany wyłącznie do synchronizacji roli „na służbie”.
Nie publikuj tokenu w repozytorium.

DISCORD_ON_DUTY_ROLE_ID
ID roli Discord nadawanej po rozpoczęciu służby i odbieranej po jej zakończeniu.
Rola bota musi znajdować się wyżej od tej roli w hierarchii Discorda.

Jeśli opcjonalnych zmiennych nie ustawisz, CAD/MDT nadal działa normalnie.

TEST LOKALNY
npm install

PowerShell:
$env:DISCORD_CLIENT_SECRET="TWOJ_SECRET"
$env:SESSION_SECRET="bardzo-dlugi-losowy-sekret-minimum-32-znaki"
$env:BASE_URL="http://localhost:3000"
npm start

Następnie otwórz http://localhost:3000 i ustaw callback:
http://localhost:3000/auth/callback

TYLKO DO LOKALNEGO TESTU BEZ DISCORDA
Możesz dodatkowo ustawić DEV_LOGIN_ENABLED=true i wejść na:
http://localhost:3000/auth/dev
Ta funkcja działa wyłącznie, gdy BASE_URL wskazuje localhost lub 127.0.0.1.

BAZA DANYCH
Domyślnie rekordy są zapisywane w ksp_mdt_data.json. Na darmowym Renderze plik może
zniknąć po ponownym wdrożeniu albo restarcie instancji. Możesz ustawić DB_FILE na
ścieżkę trwałego dysku, np. /var/data/ksp_mdt_data.json. Do większego wdrożenia
najlepszym następnym krokiem będzie PostgreSQL.
