LIBERTY OPERATIONS CENTER V3 PREVIEW — RENDER
==============================================

CO TO JEST
Obecny moduł KSP został zachowany i osadzony w większym centrum operacyjnym całego
serwera ER:LC. Wersja V3 jest wizualnym i architektonicznym podglądem przyszłego
systemu. Działające funkcje KSP/CAD nadal zapisują dane. Nowe moduły oznaczone jako
„Podgląd” nie zapisują jeszcze własnych danych.

NAJWAŻNIEJSZA ZMIANA DOSTĘPU
DISCORD_REQUIRE_ROLE=false
- użytkownik musi zalogować się Discordem,
- użytkownik musi być członkiem wskazanego serwera DISCORD_GUILD_ID,
- konkretna rola nie jest obecnie wymagana,
- wszystkie moduły podglądowe są widoczne po zalogowaniu.

Jeżeli później chcesz ponownie wymagać roli:
DISCORD_REQUIRE_ROLE=true
DISCORD_ROLE_ID=ID_ROLI_DOSTĘPOWEJ

AKTUALIZACJA ISTNIEJĄCEGO WDROŻENIA
1. Podmień pliki w tym samym repozytorium GitHub.
2. Render wykona nowe wdrożenie automatycznie.
3. Nie zmieniaj domeny usługi, jeżeli chcesz zachować obecny callback Discord.
4. W Render -> Environment dodaj:
   DISCORD_REQUIRE_ROLE=false
5. Sprawdź, czy BASE_URL (jeśli istnieje) wskazuje dokładnie obecną domenę HTTPS.
6. Callback w Discord Developer Portal musi być dokładnie:
   https://TWOJ-ADRES.onrender.com/auth/callback

WYMAGANE ZMIENNE
DISCORD_CLIENT_ID       ID aplikacji Discord.
DISCORD_CLIENT_SECRET   Sekret OAuth2. Nigdy nie publikuj go w repozytorium.
DISCORD_GUILD_ID        ID serwera Discord, którego członkostwo jest sprawdzane.
DISCORD_REQUIRE_ROLE    Ustaw false dla samego członkostwa Discord.
SESSION_SECRET          Długi losowy sekret sesji.

OPCJONALNA ROLA DOSTĘPOWA
DISCORD_ROLE_ID         Używana tylko, gdy DISCORD_REQUIRE_ROLE=true.

OPCJONALNY ADMINISTRATOR
DISCORD_ADMIN_ROLE_ID   Pozwala wykonywać operacje administracyjne, np. usuwanie.
Brak tej zmiennej nie daje automatycznie administratora zwykłym użytkownikom.

OPCJONALNE POŁĄCZENIE Z DISCORDEM
DISCORD_LOG_WEBHOOK_URL     Powiadomienia o służbie, zdarzeniach i przydziałach.
DISCORD_BOT_TOKEN           Token bota; wyłącznie w zmiennych środowiskowych.
DISCORD_ON_DUTY_ROLE_ID     Rola nadawana podczas aktywnej służby.

OPCJONALNE PRZYGOTOWANIE ER:LC
ERLC_SERVER_KEY             Klucz prywatnego serwera ER:LC.
ERLC_WEBHOOK_PUBLIC_KEY     Klucz publiczny do weryfikacji webhooków ER:LC.

W V3 PREVIEW ustawienie powyższych zmiennych jedynie pokaże gotowość integracji.
Kod nie wykonuje jeszcze żądań do ER:LC i nie ujawnia sekretów w przeglądarce.

BAZA DANYCH
Projekt zachowuje nazwę ksp_mdt_data.json, aby nie zrywać zgodności z V2.
Na darmowym Renderze lokalny plik może zniknąć po ponownym wdrożeniu lub wymianie
instancji. Do trwałego użycia potrzebny będzie PostgreSQL albo trwały dysk.

Możesz ustawić DB_FILE na ścieżkę trwałego dysku, np.:
DB_FILE=/var/data/ksp_mdt_data.json

TEST LOKALNY
npm install

PowerShell:
$env:DISCORD_CLIENT_SECRET="TWOJ_SECRET"
$env:SESSION_SECRET="bardzo-dlugi-losowy-sekret-minimum-32-znaki"
$env:BASE_URL="http://localhost:3000"
$env:DISCORD_REQUIRE_ROLE="false"
npm start

Tylko lokalnie bez Discorda:
$env:DEV_LOGIN_ENABLED="true"
Następnie otwórz:
http://localhost:3000/auth/dev

BEZPIECZEŃSTWO
Nigdy nie wysyłaj ani nie umieszczaj w ZIP-ie:
- DISCORD_CLIENT_SECRET,
- DISCORD_BOT_TOKEN,
- ERLC_SERVER_KEY,
- SESSION_SECRET,
- adresów prywatnych webhooków,
- DATABASE_URL.

System służy wyłącznie do roleplay. Nie przechowuj prawdziwych danych osobowych.


===============================================================
AWARYJNY TRYB PODGLADU — HOTFIX 3.0.1
===============================================================

Na ekranie logowania jest dodatkowy przycisk „Wejdz do podgladu”.
Pozwala on obejrzec wszystkie moduly, gdy Discord zwraca blad 429.
Tryb jest tylko do odczytu: nie zapisuje, nie edytuje i nie usuwa danych.

Domyslnie jest wlaczony. Aby go pozniej wylaczyc, dodaj na Renderze:
PREVIEW_LOGIN_ENABLED=false

Logowanie Discord pozostaje glowna metoda pelnego dostepu.
